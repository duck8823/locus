# Spec: #201 view single GitHubPr unified diff

## 目的

`cargo run -- github owner/repo#123` で GitHub PR の unified diff を Slint ウィンドウに表示する。v0.1 で「PR を開ける」状態まで到達させる。

## 実装範囲

### argv パース

`src/main.rs` に最小の argv ハンドラを足す:

- 引数なし / コマンド名 → 既存の Terminal ペインのみ (例: `cargo run`, `cargo run -- bash`)
- `github owner/repo#N` → GitHub PR を fetch して diff viewer を表示

### octocrab で PR スナップショットを取得

`src/github/pull_request.rs` に以下を追加:

```rust
pub struct PullRequestFile {
    pub file_id: FileId,
    pub file_path: String,
    pub status: FileStatus,   // Added / Modified / Removed / Renamed
    pub before_content: Option<String>,
    pub after_content: Option<String>,
    pub patch: Option<String>,
    pub is_binary: bool,
}

pub struct PullRequestSnapshot {
    pub target: ReviewTarget,   // GitHubPr variant
    pub title: String,
    pub head_sha: String,
    pub base_sha: String,
    pub files: Vec<PullRequestFile>,
}

pub async fn fetch_pr_snapshot(
    client: &octocrab::Octocrab,
    owner: &str,
    repo: &str,
    pr_number: u64,
) -> Result<PullRequestSnapshot, GithubError>
```

- PR file list を octocrab で取得
- 各ファイルについて before/after content を `GET /repos/{owner}/{repo}/contents/{path}?ref=<sha>` で取得
  - added ファイルは before が None
  - removed ファイルは after が None
  - binary ファイルは `is_binary=true`, content は None
  - 大きいファイル（Content-Length / base64 decode 失敗等）は patch_missing として扱う

### authentication

- `GITHUB_TOKEN` / `GH_TOKEN` 環境変数から読む
- どちらも無ければ unauthenticated（public repo のみ動作、レート制限あり）

### diff 正規化

`src/review/diff.rs` を拡張するか、新規 `src/review/diff_builder.rs` で:

```rust
pub fn build_file_diff(
    before: Option<&str>,
    after: Option<&str>,
    file_id: FileId,
    file_path: String,
) -> FileDiff
```

シンプルな Myers diff or LCS で十分。`similar` crate を追加してもよい。

当面は行比較ベースで context=3 の unified diff を生成する。

### unsupported 処理

binary / patch_missing / content 取得失敗 のいずれも `UnsupportedFile` として記録し、UI に明示表示する。

### Slint UI

`ui/app.slint` を拡張:

- 上部: タイトル + head SHA + base SHA badge
- 左ペイン: ファイル一覧（サイドバー）
- 右ペイン: 選択中ファイルの diff（色付き unified diff）
- 既存の Terminal ペインは一旦非表示 or 下部タブ扱いにする（ここで Terminal を同時表示しない）

実装判断: v0.1 で Terminal ペインと diff viewer を同画面に並べるかは別問題。Sprint 初期では **別モード**（起動時のサブコマンドで切り替え）で十分。

- `cargo run` / `cargo run -- bash` → Terminal モード（既存の PoC）
- `cargo run -- github owner/repo#N` → Diff viewer モード

### 非同期と Slint の統合

- tokio runtime を `main` で立てて、PR fetch は tokio 上で実行
- 取得結果を `slint::invoke_from_event_loop` で UI スレッドに戻してモデルに書き込む

## 非スコープ

- ファイル lazy load の最適化（単純 eager fetch で OK、大きな PR は後続対応）
- hunk folding の UI（内部モデルは folding 可能な構造にしておく）
- selection ハンドリング（#202）
- PR 一覧サイドバー（#204）
- Terminal ペインと diff viewer の同時表示（後続）

## テスト

- `build_file_diff` のユニットテスト（before/after から Hunk 構造が出ること、Context/Added/Removed の分類が正しいこと）
- argv パーサのユニットテスト（`github owner/repo#123` の分解）
- unsupported file の判定（binary / patch missing）

## 検証

- `cargo build` / `cargo clippy --all-targets -- -D warnings` / `cargo test`
- 手動: `cargo run -- github duck8823/locus#209` で実際に diff が見えること（ローカル検証はユーザー手元）

## 注意

- `tokio` を rt-multi-thread で使うと bin 全体が tokio context になる。Slint のイベントループは別スレッドで回したいので、main を sync にしつつ、fetch だけ tokio::runtime::Runtime::block_on で呼ぶ形が安全。
- octocrab の現行 API は async 前提。`Octocrab::builder().personal_token(...)` を使う。

## 実装ステップ

1. Cargo.toml に octocrab, tokio, similar を追加
2. `src/github/pull_request.rs` で octocrab 呼び出しと PullRequestSnapshot 型を作る
3. `src/review/diff_builder.rs` で build_file_diff を作る（similar crate 使用）
4. Slint UI に diff viewer コンポーネントを追加
5. `src/main.rs` に argv パーサと diff viewer モード起動を追加
6. テスト追加
7. 検証
